## 1. 프로젝트 개요

### 1-1. 목표
SNS(틱톡, 인스타그램) 화면녹화 환경에서 유통되는 딥페이크 영상을 실시간으로 탐지하는 웹 서비스 개발

### 1-2. 핵심 문제 정의
기존 딥페이크 탐지 연구는 FaceForensics++, DFDC 등 고화질 원본 영상 기준으로 AUC 0.99 이상을 달성하지만, 실제 SNS 환경에서는 성능이 급격히 하락한다.

- SNS 화면녹화 과정에서 발생하는 압축 블러, 인코딩 아티팩트, 해상도 저하
- 기존 모델을 SNS 영상에 그대로 적용 시 AUC **0.411** (동전던지기 수준) 확인
- → 실사용 환경 특화 탐지 모델 필요

### 1-3. 차별점
공개 벤치마크 수치에 의존하지 않고, **직접 SNS OOD 데이터셋 140개를 수집**하여 실제 배포 환경 기준으로 평가.

## 3. 개발 과정

### 3-1. 데이터셋 구성

| 데이터셋 | 용도 | 규모 |
|---------|------|------|
| FaceForensics++ (FF++) | 학습/평가 | 600개 (eval) |
| DFDC | 학습/평가 | 600개 (eval) |
| DF40 | 학습 보조 | 243,556개 (splits_v35) |
| **SNS 자체수집 (OOD)** | **핵심 평가** | **140개 (real 70 / fake 70)** |

**SNS OOD 데이터셋 수집 방식**
- 틱톡/인스타그램에서 딥페이크 영상 및 실제 인물 영상 시청 후 맥북 화면녹화
- 3차에 걸쳐 수집 (2026-05-12, 05-14, 05-20)
- 팀이 직접 라벨링 (영상 출처 + 육안 검토)
- 기존 공개 데이터셋으로는 측정 불가능한 SNS 도메인 갭을 정량화하기 위한 독자적 벤치마크

### 3-2. 모델 개발 이력 요약

총 37버전에 걸친 실험을 진행했으며, 크게 세 단계로 구분된다.

**Phase 1 — 아키텍처 탐색 (v1~v18)**

FF++/DFDC 기반으로 Xception, F3NetLite, Freq-only 아키텍처를 실험하며 in-distribution 성능을 최적화했다.
- v7: FF++/DFDC 통합 학습으로 cross-domain gap 해소 (0.277 → ~0)
- v8: FAD 3-band + SEBlock Fusion 도입, FF++ 0.9737 / DFDC 0.9881
- v11~v13: DF40 단독 학습 시 FF++ cross-eval AUC 0.49로 붕괴 → DF40 단독 학습 실패 결론
- v15+v16+v17 앙상블: FF++ 0.9995, DFDC 0.9980 달성 → **SNS 0.411 확인, Phase 1 한계 선언**

**Phase 2 — SNS 특화 (v19~v36)**

F3NetLiteV20(RGB+주파수 이중 분석) 아키텍처를 기반으로 SNS 도메인 특화 학습을 진행했다.
- v20: DF40+SNS 통합 학습 기본 베이스 모델 구축
- v24: SNS real 5x 오버샘플 파인튜닝 → OOD **0.872** (단일 모델 최고)
- v28: DINOv2+Xception 차원 불균형 실패
- v30: EfficientNet-B4 단일 백본 시도 → 기존 앙상블 미달
- v34: DINOv2-large frozen + MLP → OOD 0.806 (Transformer 계열 첫 의미있는 성과)

**Phase 3 — 앙상블 최적화**

75가지 이상 조합 sweep 및 하이퍼파라미터 튜닝을 통해 최적 앙상블 구성을 도출했다.

### 3-3. 주요 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| DFDC AUC 0.686 (v6) | FF++ only 학습으로 도메인 일반화 미흡 | FF++/DFDC 통합 학습 (v7) |
| DF40 학습 후 FF++ cross-eval 0.491 (v11) | DF40 도메인 과적합 | DF40 단독 학습 폐기, 혼합 전략 채택 |
| SNS OOD 0.411 (Phase 1 앙상블) | 화면녹화 압축/블러 도메인 갭 | SNS 특화 데이터 수집 + F3NetLiteV20 |
| v33 성능 하락 (OOD 0.737) | backbone frozen + 과도한 SNS 오버샘플 → Catastrophic forgetting | 오버샘플 비율/프리징 전략 조정 |
| cuDNN 심볼 충돌 RuntimeError | 시스템 cuDNN vs PyTorch 번들 충돌 | `torch.backends.cudnn.enabled = False` 전 스크립트 적용 |

## 4. 최종 성과

### 4-1. 모델 성능

| 구분 | 구성 | SNS OOD AUC |
|------|------|-------------|
| Phase 1 앙상블 (기준점) | v15+v16+v17 | 0.411 |
| 단일 모델 최고 | v24 | 0.872 |
| **최종 앙상블** | **v24(20%)+v25(50%)+v29(30%)** | **0.8898** |

**평가 프로토콜**: CROP_SCALE=1.0, first5 프레임, Mean 집계, OOD 140개 기준

### 4-2. 왜 OOD 수치를 신뢰하는가

기존 연구들은 in-distribution 테스트셋 기준 AUC 0.99 이상을 발표하지만, 동일한 방식으로 평가하면 우리 모델도 AUC 1.0에 가까운 수치가 나온다. 우리는 이 수치를 제출 지표로 삼지 않았다.

실제 서비스 환경(SNS 화면녹화)을 직접 재현한 140개 OOD 데이터셋을 구축하고, 거기서 측정한 **0.8898**이 실제 배포 성능을 대표한다고 판단했다.

## 5. 한계점 및 오류 분석

### 5-1. 오분류 케이스 분석

OOD 140개 중 오분류된 영상을 분석한 결과:

- **Fake Missed (탐지 실패)**: 15건 — 모델 점수가 낮게 나온 가짜 영상. 주로 SNS 특유의 과도한 압축으로 딥페이크 흔적이 소실된 경우
- **False Alarm (오탐)**: 12건 — 진짜 영상을 가짜로 판정. 뷰티 필터, 조명 변화 등 주파수 이상이 발생한 실제 영상

### 5-2. 한계

- SNS OOD 데이터셋 140개는 수동 수집으로 규모에 한계 있음
- WildDF, DFDC 등 공개 데이터셋 기준 성능은 0.55~0.62 수준 (SNS 특화의 trade-off)
- v14, v19~v23, v31 구간 실험 기록 부재

## 6. 결론

SNS 화면녹화 환경이라는 실사용 도메인에 특화된 딥페이크 탐지 시스템을 구축했다. 37버전에 걸친 실험을 통해 F3NetLiteV20(RGB+주파수 이중 분석) 기반 3-way 앙상블이 SNS OOD에서 가장 효과적임을 확인했으며, 직접 수집한 140개 OOD 벤치마크에서 AUC 0.8898을 달성했다.

공개 벤치마크의 과적합 수치를 맹신하지 않고 실제 배포 환경 기준으로 일관되게 평가한 점이 본 프로젝트의 핵심 기여다.
