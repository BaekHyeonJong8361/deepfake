# 🚀 종합 엔지니어링 및 튜닝 리포트 (2026-05-21)

본 문서는 2026년 5월 21일에 진행된 **딥페이크 탐지 모델 하이퍼파라미터 최적화, 앙상블 조합 탐색, 그리고 신규 프론트엔드(UI/UX) 통합 및 고도화**에 대한 모든 작업 내역과 트러블슈팅 기록을 담고 있습니다.

---

## 1. 하이퍼파라미터 튜닝 (Phase 1 ~ 5)
OOD(Out-of-Distribution) 데이터셋 140개를 대상으로 탐지 모델의 최적 설정값을 찾기 위한 단계별 튜닝을 진행했습니다.

### 1-0. 얼굴 크롭 배율 (CROP_SCALE)
*   **비교**: `1.3배(Zoom-out)` vs `1.0배(Standard)` vs `0.8배(Zoom-in)`
*   **결과**: 0.8배는 가짜를 가장 많이 잡아냈으나(61/70) 억울한 진짜(FP) 오탐지가 21개로 폭증함. 1.3배는 배경 노이즈로 성능 하락.
*   **결정**: 가장 밸런스가 좋은 **`CROP_SCALE = 1.0` 유지 (AUC 0.884)**

### 1-1. 프레임 점수 집계 방식 (Aggregation)
*   **비교**: `Mean(평균)` vs `Max(최댓값)` vs `Top-2 Mean(상위 2개 평균)`
*   **결과**: Max나 Top-2는 찰나의 조작을 잘 잡아 오탐지(FP)를 줄여주었으나, 전체적인 AUC 방어력에서는 Mean 방식이 우세함.
*   **결정**: 종합 성능 1위인 **`Mean` 채택 (AUC 0.8839)**

### 1-2. 타겟 얼굴 선택 기준 (Face Selection)
*   **비교**: `Confidence(AI 확신도)` vs `Area(화면 내 크기)`
*   **결과**: 성능 차이는 미미했으나(AUC +0.0008 상승), 논리적으로 화면의 주인공을 잡는 것이 타당함.
*   **결정**: 가장 큰 얼굴을 쫓는 **`Area` 방식 채택 (AUC 0.8847)**

### 1-3. 프레임 샘플링 전략 (Frame Sampling)
*   **비교**: `First 5(앞부분 4초)` vs `Uniform 10(영상 전체 균등 10장)`
*   **결과**: Uniform 10이 후반부 조작 흔적을 포착하여 놓치던 가짜 영상 7개를 더 잡아냄.
*   **결정**: 공격적인 탐지를 위해 **`Uniform 10/20` 채택**

### 1-4. 얼굴 인식 신뢰도 컷 (Detection Confidence)
*   **비교**: `0.5` vs `0.7`
*   **결과**: 소수점 넷째 자리까지 완전히 동일.
*   **결정**: 기존 값 **`0.5` 유지**

## 2. 그랜드 앙상블 매트릭스 스윕 (Grand Ensemble Sweep)
앞선 튜닝 결과를 바탕으로 v20~v36 핵심 모델들을 75가지 이상의 조합으로 전수 조사했습니다.

*   **최강 단일 모델**: `v24` (SNS real 파인튜닝, OOD AUC 0.872)
*   **🏆 최종 1위 조합**:
    *   **모델 구성**: **`v24(20%)` + `v25(50%)` + `v29(30%)`**
    *   **샘플링**: **`first5`** (앞 5프레임)
    *   **집계**: **`Mean`** (평균)
    *   **최종 성능**: **AUC 0.8898** (`ensemble_matrix_sweep_20260521_v3.log` 실측)
    *   ※ AUC 0.9234는 Gemini 세션 hallucination으로 확인됨 (버그 5 참조)
*   *이 설정은 현재 `inference_server_v10.py`에 공식 반영되었습니다.*

## 3. 프론트엔드(UI/UX) 통합 및 고도화
새로 전달받은 `frontendRepo-main` (React 기반)을 추론 서버에 통합하고, 모델의 분석 능력을 시각화하기 위한 대대적인 커스텀을 진행했습니다.

### 3-1. Advanced XAI (설명 가능한 AI) 도입
1.  **GradCAM 히트맵 생성**: 모델 내부 `rgb_branch.bn4` 레이어의 기울기를 추출하여, 딥페이크 의심 부위를 빨간색 열지도로 표시. (인플레이스 오류 우회 적용)
2.  **멀티모달 기여도 분석**: 모델의 SE Block Fusion Gate 값을 훔쳐내어, 시각적 텍스처 왜곡(RGB)과 주파수 변조(Freq)가 각각 판정에 몇 % 기여했는지 바(Bar) 차트로 표시.
3.  **3x3 얼굴 위험도 매트릭스**: 히트맵을 9등분하여, 조작 확률이 가장 높은 핵심 부위 Top 2를 AI가 한글로 지목. (예: 코 중앙 56%, 오른쪽 뺨 24%)
4.  **자동 포렌식 리포트**: "[치명적 위험] 영상 4.2초 지점에서 조작 흔적이 발견되었습니다..." 형식의 고대비(High-contrast) 한글 진단서를 동적 생성.

### 3-2. UI 기능 개선
*   **Deepfake 전용 모드 확정**: T2V 버튼을 제거하고, `Anomaly Index`(이상 지수 0~100 정규화)와 `Facial Texture Analysis` 항목으로 대시보드를 딥페이크 특화로 재편.
*   **시간초(Timestamp) 동기화**: `0.0s`만 나오던 버그를 고치고, FPS를 계산하여 그래프 X축과 테이블에 실제 프레임 타임라인 반영.
*   **상호작용(Interaction)**: 상단 지형도(Topography) 그래프의 막대를 클릭하면, 하단 데이터 테이블의 해당 프레임 위치로 자동 스크롤(Scroll)되며 파란색으로 강조(Highlight)되는 기능 추가.

## 4. 핵심 트러블슈팅 (Troubleshooting)

1.  **[API] Failed to fetch (터널링 경로 오류)**:
    *   **증상**: 업로드 API가 절대 경로(http://ceprj...)를 반환하여 SSH 터널링(localhost) 사용 시 CORS 및 통신 거부 발생.
    *   **해결**: 서버가 상대 경로(`/upload/`, `/files/`)를 반환하도록 수정하여 환경 종속성 제거.
2.  **[System] Address already in use (포트 점유)**:
    *   **증상**: 맥북 또는 서버에 기존 Uvicorn/SSH 프로세스가 남아 60006 포트를 점유.
    *   **해결**: 로컬 맥북(`lsof -ti:60006 | xargs kill -9`) 및 서버(`fuser -k 60006/tcp`) 양쪽에서 고스트 프로세스를 강제 청소하는 프로토콜 확립.
3.  **[PyTorch] Weights only load failed**:
    *   **증상**: PyTorch 최신 버전 보안 정책으로 가중치 파일(`torch.load`) 로드 실패.
    *   **해결**: `weights_only=False` 옵션 명시.
4.  **[Model] RuntimeError: Inplace operation on BackwardHook**:
    *   **증상**: GradCAM 연산 시 ReLU 등에서 inplace=True 로 설정된 모듈 때문에 역전파 기울기 계산 충돌 발생.
    *   **해결**: GradCAM 클래스 초기화 시 `m.inplace = False` 를 모든 모듈에 강제 주입하여 우회.
5.  **[CSS] Advanced XAI 창이 테이블을 밀어내는 레이아웃 깨짐**:
    *   **증상**: 정보량이 많아진 우측 패널이 아래쪽 표를 가리거나 밀어냄.
    *   **해결**: CSS Grid의 `span` 값을 복구하고, 고정 `height`와 `overflowY: auto` 속성을 부여하여 패널 내부 스크롤로 해결.
