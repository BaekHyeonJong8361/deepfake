# 개요 및 문제 배경

**문제 배경**
- **SNS 딥페이크 확산** — 인스타·틱톡에 실제로 퍼져 사람들이 속음
- **벤치마크 ≠ 현실** — 학회 데이터는 풀렸지만 실제 SNS 영상은 도메인이 다름
- **점수만으론 신뢰 불가** — "92% 가짜" 숫자만으론 왜 가짜인지 알 수 없음
- **진화하는 생성기** — 최신 생성기는 기존 탐지기를 우회

**해결방안** — Explainable SNS Detection
- SNS 도메인 특화 탐지 모델 구현 (F3NetLiteV20)
- 실제 SNS 환경 데이터로 학습 (DF40 + SNS real)
- 실제 SNS 화면녹화로 **OOD 일반화 성능 검증**
- **XAI 설명 병행 제공** — 왜 가짜인지 근거까지

> 발표자 노트: 딥페이크는 이미 SNS에서 사람들이 속는 현실 문제입니다. 그런데 학회 데이터는 잘 풀려도 실제 SNS는 도메인이 달라 안 잡히고, 점수만으론 왜 가짜인지 신뢰하기 어렵습니다. 그래서 저희는 SNS 환경에 특화된 모델을 만들고, 실제 SNS 영상으로 일반화를 검증하고, 판정 근거를 XAI로 함께 제공하는 걸 목표로 잡았습니다.

---

# 벤치마크는 풀렸는데, 현실은?

- 학회 벤치마크에서는 거의 완벽한 성능
- FF++ AUC **0.97** / DFDC AUC **0.99**
- 그러나 우리가 풀고 싶은 건 학회 데이터가 아니라, 실제 인스타·틱톡에 올라오는 딥페이크
- 핵심 질문: **학회 데이터는 잘 잡는데, 실제 SNS 영상도 잡을까?**

> 발표자 노트: 딥페이크 탐지는 학회 데이터셋에선 거의 풀린 문제처럼 보입니다. 저희 모델도 FF++ 0.97, DFDC 0.99가 나왔으니까요. 그런데 진짜 풀고 싶었던 건 실제로 사람들이 속는 SNS 영상이었습니다. 과연 이 모델이 그걸 잡을 수 있을까요?

---

# 접근: 모델 아키텍처

**F3NetLiteV20 (dual-branch, ~44M)**
- RGB Branch (Xception) → 공간 텍스처
- FAD 3-band 주파수 Branch (Xception) → 주파수 아티팩트
- SEBlockFusion (채널 어텐션)으로 두 갈래 융합 → 판정

**추론 흐름** (AI 파트)
- 영상 → 얼굴 검출·크롭 → 프레임별 추론 → 앙상블 판정 → XAI 근거 생성

> 발표자 노트: 저희 모델은 두 갈래로 영상을 봅니다. RGB로 공간 텍스처를, 주파수 분석으로 눈에 안 보이는 합성 아티팩트를 보고, 어텐션으로 융합합니다. 영상이 들어오면 얼굴을 크롭해 프레임별로 추론하고, 앙상블로 판정한 뒤 설명 근거까지 만듭니다.

---

# 데이터 & 학습 전략

- **DF40**: 최신 생성기 40종, 약 24만 장 (sd2.1/DiT/SiT 등 최신 딥페이크)
- **FF++ / DFDC real**: 고전 진짜 얼굴
- **SNS real**: 실제 SNS 화면녹화, 자체 수집
- **SNS 도메인 증강**: 화면녹화 아티팩트 모사 (Double JPEG, bilateral/median 스무딩, resize-rescale, 노이즈)
- 핵심: 최신 가짜 + 실제 SNS 화면 환경을 학습에 직접 주입
- 학습 데이터와 평가셋(140개)은 **출처가 분리됨**

> 발표자 노트: 데이터가 전략의 핵심입니다. 가짜는 최신 생성기 40종으로 만든 24만 장 DF40, 진짜는 기존 데이터셋과 함께 실제 SNS 화면녹화를 직접 모았습니다. 화면녹화 특유의 압축·블러도 증강으로 모사해 모델이 SNS 환경을 미리 겪게 했습니다.

---

# 잘 잡는 줄 알았는데, SNS는 동전던지기였다

- in-distribution: FF++ **0.97** / DFDC **0.99**
- SNS 화면녹화 OOD: **0.41**
- 0.5(랜덤)보다도 낮음 — 도메인이 바뀌면 성능 붕괴
- **벤치마크 잘 잡는 모델 ≠ 실제 SNS 잡는 모델**
- 이 0.41을 끌어올리는 것이 프로젝트의 진짜 시작

> 발표자 노트: 결과는 충격이었습니다. 0.97, 0.99를 찍던 모델이 실제 SNS 영상에선 0.41, 동전던지기보다 못했습니다. 이 0.41을 어디까지 올릴 수 있는가가 진짜 시작이었습니다.

---

# 0.41을 풀려고: 37번의 실험

- **v1~v5** 아키텍처 탐색 (Xception → F3NetLite → Freq-only)
- **v6~v7** FF++/DFDC 통합 → cross-domain gap 해소
- **v8~v10** 주파수 dual-branch(F3NetLite) 도입
- **v11~v13** DF40 단독 학습 → in-dist 과적합 실패
- **v15~v17** SNS 증강 투입 → 그래도 OOD 0.41 (Phase1 실패 선언)
- **v20** DF40+SNS 첫 학습 → 0.67 (base 모델)
- **v24** SNS real 5x 파인튜닝 → **0.87 (단일 최고)**
- **v28/v32/v34** DINOv2 계열 시도 → 0.81 (못 넘음)
- **sweep** v24+v25+v29 앙상블 → 0.89
- 총 37개 버전 — 변수를 하나씩만 바꾸며 통제 실험

> 발표자 노트: 이 0.41을 풀려고 버전을 하나씩만 바꿔가며 총 37번 실험했습니다. 아키텍처 탐색부터 데이터 교체, 파인튜닝, 앙상블까지 — 그중 뭐가 진짜 효과였는지를 다음 장에서 정리했습니다. (5초 안에 빠르게 넘김)

---

# 무엇이 OOD를 풀었나 (Ablation)

- **Phase1** (주파수 모델 + SNS 증강, DF40 없음): **0.41**
- **+DF40 데이터** (fake를 DF40 24만/40종으로 교체): **0.67** (+0.26)
- **+SNS real 파인튜닝** (실제 SNS real 5x 오버샘플): **0.87** (+0.20)
- 결론: OOD를 끌어올린 주역은 화려한 구조가 아니라 **데이터 전략** (주파수 구조 기여는 +0.066, 보조)
- *(작게)* v16→v20은 구조 사실상 동일 → 바뀐 변수는 **학습 데이터뿐**. 주파수는 개별 판정 근거론 강함(슬8) — 다른 층위

> 발표자 노트: 변수를 하나씩만 바꿔 통제 실험했습니다. 흔히 '주파수가 핵심'이라는데, 저희 데이터는 아니었습니다. 주파수 모델에 SNS 증강만 넣어도 0.41이었거든요. 진짜 레버는 둘 — DF40으로 교체하니 0.67, SNS 영상으로 파인튜닝하니 0.87. 답은 화려한 구조가 아니라 도메인에 맞는 데이터 전략이었습니다. (ablation_ood.png 이미지 삽입)

# 왜 최신 모델이 아니라 이걸 썼나 + 정직한 한계

**왜 F3NetLiteV20?**
- DINOv2-large (최신 SSL) OOD = 0.81
- F3NetLiteV20 (우리) OOD = 0.87 → 우리 OOD에선 최신 backbone을 이김 + 추론 가벼움

**최종 성능 (정직하게)**
- 단일 모델 v24 = **0.87 ← 대표 수치**
- 앙상블 = 0.89 (단일 대비 +0.018 보조, 동일 셋 sweep이라 낙관적)

**한계 (먼저 인정)**
- 평가셋 n=140 — 라벨 신뢰성 위해 양보다 검증된 데이터 택함, 신뢰구간 넓음 인정
- 강한 후보정 오탐 — 보정·필터가 극단적으로 강한 진짜 영상은 딥페이크로 오탐 가능

**개선사항**
- 검증된 OOD 데이터 확장 → 신뢰구간 좁히고 현재 0.87을 더 위로
- 보정 강도 분리 학습 → "심한 보정 ≠ 딥페이크" 오탐 감소
- 최신 생성기 지속 편입 → 진화하는 딥페이크 대응

> 발표자 노트: 최신 DINOv2도 다 해봤지만 우리 OOD에선 0.81로 더 낮았고 추론도 무거웠습니다. 최종 성능은 정직하게 단일 0.87. 앙상블 0.89는 같은 셋에서 튜닝한 값이라 보조로만 봅니다. 평가셋 140개는 크롤링 대량수집 대신 AI생성 여부를 직접 검증한 결과라, 신뢰구간이 넓은 점 인정하고 데이터 확장을 다음 과제로 둡니다. 또 보정·필터가 극단적으로 강한 진짜 영상은 오탐하기도 해서 보정 강도 분리를 개선점으로 둡니다. 저희는 이걸 0.41에서 0.87로 끌어올린 현실적 출발점으로 봅니다.

# XAI: 점수가 아니라 '왜 FAKE인지'를 보여준다

- **판정**: CRITICAL / FAKE, Deepfake 확률 92.5%, 분석 프레임 30
- **어디**: Grad-CAM 히트맵 — 코·얼굴 중앙에 집중
- **언제**: Frame-by-Frame Anomaly 그래프 + threshold 라인
- **무엇으로**: 영상마다 RGB(공간) vs 주파수 중 결정 근거가 다름 → 매 판정마다 비율 분해 (예시 이 영상: Freq 63% / Spatial 37%)
- 공간·시간·주파수 **3중 근거 = 설명 가능한 포렌식 증거**, 고정 비율 아닌 영상별 적응형 설명
- 포지셔닝: 과학적 주장이 아니라 **제품 UX** — 기자·일반인이 눈으로 납득

> 발표자 노트: 대부분 탐지기는 '92% 가짜' 숫자만 던지지만 본 사람은 '왜?'를 못 얻습니다. 저희는 어디(Grad-CAM)·언제(프레임 그래프)·무엇으로(기여도) 3중 근거를 동시에 줍니다. 여기서 중요한 건, 저희 모델은 영상마다 공간 텍스처와 주파수 중 무엇이 더 결정적인지를 다르게 봅니다. 지금 이 영상은 주파수 단서가 63%로 더 컸던 경우고, 다른 영상은 공간 쪽이 더 클 수도 있습니다. 고정된 비율이 아니라 매 판정마다 '이번엔 무엇 때문에 가짜로 봤는지'를 분해해 보여주는 겁니다. 'OOD를 푼 건 데이터 전략'과는 다른 층위고요. 사용자가 숫자가 아니라 눈으로 납득하게 하는 제품 경험으로 봅니다. (XAI 제품 캡처 이미지 삽입)

# 정리

- 학회에선 풀린 듯 보이던 문제가 실제 SNS에선 **0.41**
- 데이터 전략(DF40 + SNS real 파인튜닝)으로 **0.87**까지
- 그 판정에 '왜'라는 **설명(XAI)**을 붙였다
- 한계는 분명히 인정하되, 실제로 쓸 수 있는 현실적 baseline을 만들었다

> 발표자 노트: 정리하면, 학회에선 풀린 듯한 문제가 SNS에선 0.41이었고, 데이터 전략으로 0.87까지 끌어올린 뒤 판정에 설명을 붙였습니다. 한계는 인정하지만 실제로 쓸 수 있는 현실적 baseline을 만들었습니다. 감사합니다.
