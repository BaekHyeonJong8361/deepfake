## 3. 핵심 모델 구조: F3NetLiteV20

### 3-1. 설계 배경

SNS 화면녹화 딥페이크는 육안으로 구별하기 어렵다. 그러나 GAN/Diffusion 합성 과정에서 발생하는 **주파수 이상**은 압축 후에도 일부 잔존한다. RGB 시각 정보와 주파수 분석을 결합한 이중 분석이 핵심 전략이다.

### 3-2. 네트워크 구조

```
입력: 299×299 RGB 이미지
        │
        ├──────────────────────────────────────────┐
        │                                          │
   [RGB Branch]                            [FAD Freq Branch]
   Xception backbone                        FFT 변환
   (ImageNet pretrained)                    → 3개 주파수 대역 분리 (저/중/고)
   출력: 2048-d 특징                         → Xception backbone (in_chans=9)
                                            출력: 2048-d 특징
        │                                          │
        └──────────────┬───────────────────────────┘
                       │
              [SEBlockFusion]
              BatchNorm1d (각 branch)
              → Concat: 4096-d
              → SE Channel Attention (가중치 자동 학습)
              → fc_head: Linear(4096 → 1)
              → Sigmoid
                       │
              딥페이크 확률 (0.0 ~ 1.0)

파라미터: ~44M
```

### 3-3. FAD (Frequency-domain Analysis Decomposition)

```python
# 주파수 대역 분리
fft = torch.fft.rfft2(x)                    # 2D FFT
magnitude = torch.abs(fft)                   # 진폭 스펙트럼
# num_bands=3 구간으로 분리 (저/중/고주파)
# add_on_filters: 학습 가능 파라미터로 대역 경계 자동 최적화
```

### 3-4. 학습 설정 (최종 앙상블 구성 모델)

| 항목 | v24 | v25 | v29 |
|------|-----|-----|-----|
| 베이스 | v20 파인튜닝 | v20 파인튜닝 | from scratch |
| SNS 오버샘플 | 5x | 2x | - |
| Learning Rate | 1e-5 | 1e-5 | 1e-4 |
| backbone frozen | O | O | X |
| 최적 val AUC | 1.000 | 1.000 | 1.000 |
| **OOD AUC** | **0.872** | **0.842** | **0.821** |
| 앙상블 가중치 | 20% | 50% | 30% |

## 5. XAI (설명 가능한 AI)

### 5-1. GradCAM

모델 내부 `rgb_branch.bn4` 레이어의 기울기를 추출하여 딥페이크 의심 부위를 히트맵으로 시각화.

```
모델 추론
    ↓
rgb_branch.bn4 레이어 그래디언트 추출
    ↓
공간적 중요도 맵 생성 (14×14 → 299×299 업샘플)
    ↓
원본 프레임에 오버레이 (빨강=고위험, 파랑=저위험)
    ↓
base64 인코딩 → 프론트엔드 전달
```

### 5-2. SE Attention 분석

SEBlockFusion의 채널 게이트 값을 추출하여 RGB 기여도 vs 주파수 기여도 비율을 시각화.
