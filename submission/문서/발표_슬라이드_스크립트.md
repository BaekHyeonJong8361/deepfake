## 슬라이드 3 — 접근: 모델 아키텍처

> 시스템 구조(프론트/백엔드/인프라)는 아키텍처 담당(김지혁) 파트라 여기선 모델만.

**[화면 묘사]**
- **모델 아키텍처 F3NetLiteV20** (도식)
  ```
  입력 (299×299 RGB)
    ├─► RGB Branch (Xception, 2048-d)         ← 공간 텍스처
    └─► FAD 3-band 주파수 Branch (Xception, 2048-d)  ← 주파수 아티팩트
            ↓ SEBlockFusion (채널 어텐션)
            ↓ Linear(4096→1) → sigmoid
    파라미터 ~44M
  ```
- **추론 흐름 (AI 파트)**: 영상 → 얼굴 검출·크롭 → 프레임별 추론 → 앙상블 판정 → XAI 근거 생성

**[스크립트]**
> "저희 모델은 두 갈래로 영상을 봅니다. 하나는 일반 RGB로 얼굴의 공간적 텍스처를, 다른 하나는 주파수 분석으로 사람 눈에 안 보이는 합성 아티팩트를 봅니다. 이 둘을 어텐션으로 융합해 판정하는 F3NetLiteV20 구조입니다. 영상이 들어오면 얼굴을 크롭해 프레임별로 추론하고, 앙상블로 판정한 뒤 설명 근거까지 만듭니다."

## 슬라이드 5.5 — "0.41을 풀려고: 37번의 실험" (5초 넘김용 경로 슬라이드)

> 목적: 다 읽히는 게 아니라 **"이만큼 체계적으로 갈았다"가 한눈에 꽂히는** 슬라이드. 멘트 2문장 후 바로 넘김.

**[화면 묘사]** — 가로 타임라인/경로 형태, 단계별 칩(chip)만

```
v1 ─ v5      v6 ─ v18              v19 ─ v36                 sweep
─────────►  ─────────────────►   ─────────────────────►   ───────►
아키텍처 탐색  in-dist 최적화        SNS 특화 파인튜닝            앙상블
Xception →   FF++/DFDC 통합 →      F3NetLiteV20 +            v24+v25+v29
F3NetLite →  주파수+증강 →          DF40 데이터 +
Freq-only    "SNS OOD 0.41 발견"   SNS real 파인튜닝
                  ▲ 벽            "단일 최고 0.87(v24)"
```

- 각 구간 핵심 한 줄(칩):
  - **v6~v7**: FF++/DFDC 통합 → cross-domain gap 해소 (in-dist)
  - **v8~v10**: 주파수 dual-branch(F3NetLite) 도입
  - **v11~v13**: DF40 단독 학습 → in-dist 과적합 실패
  - **v15~v17**: SNS 증강 투입 → 그래도 OOD 0.41 (**Phase1 실패 선언**)
  - **v20**: DF40+SNS 첫 학습 → 0.67 (base 모델)
  - **v24**: SNS real 5x 파인튜닝 → **0.87 (단일 최고)**
  - **v28/v32/v34**: DINOv2 계열 시도 → 0.81 (우리 못 넘음)
  - **sweep**: v24+v25+v29 앙상블 → 0.89
- 하단 띠: **"총 37개 버전 — 변수를 하나씩만 바꾸며 통제 실험"**

## 슬라이드 7 — "왜 최신 모델이 아니라 이걸 썼나 + 정직한 한계"

**[화면 묘사]**
- *왜 F3NetLiteV20?*
  - DINOv2-large(최신 SSL) OOD = **0.81**
  - F3NetLiteV20(우리) OOD = **0.87** → 우리 OOD에선 최신 backbone을 이김 + 추론 가벼움
- *최종 성능 (정직하게)*
  - **단일 모델 v24 = 0.87 ← 대표 수치**
  - 앙상블 = 0.89 (단일 대비 **+0.018 보조**, 단 동일 셋 sweep이라 낙관적)
- *한계 (먼저 인정)*
  - 평가셋 **n=140** — 라벨 신뢰성 위해 양보다 검증된 데이터 택함, 신뢰구간 넓음 인정
  - **강한 후보정 오탐** — 보정·필터가 극단적으로 강한 진짜 영상은 딥페이크로 오탐 가능
- *개선사항*
  - 검증된 OOD 데이터 확장 → 신뢰구간 좁히고 현재 0.87을 더 위로
  - 보정 강도 분리 학습 → "심한 보정 ≠ 딥페이크" 오탐 감소
  - 최신 생성기 지속 편입 → 진화하는 딥페이크 대응

**[스크립트]**
> "왜 요즘 유행하는 DINOv2 같은 최신 모델을 안 썼냐고 하실 수 있는데, 저희도 다 해봤습니다. 그런데 저희 OOD에선 DINOv2가 0.81, 저희 주파수 모델이 0.87로 오히려 더 좋았고 추론도 가벼웠습니다. 최종 성능은 정직하게, 단일 모델 기준 0.87입니다. 앙상블로 0.89까지 가지만, 이건 평가셋과 같은 데이터에서 가중치를 튜닝한 값이라 다소 낙관적이라 보조 지표로만 봅니다.
> 평가셋이 140개인 건 한계가 맞습니다. 다만 처음엔 크롤링으로 대량 수집했는데, 그 영상이 진짜 AI 생성인지 검증이 안 되면 점수 자체가 의미 없어지더라고요. 그래서 양보다 **라벨 신뢰성**을 택해, real 70·fake 70을 한 장씩 직접 검증한 140개로 갔습니다. n이 작아 신뢰구간은 넓다는 점, held-out 분리를 못 한 점 모두 인정하고, 데이터 확장을 명확한 다음 과제로 둡니다. 0.87은 약 13%를 놓친다는 뜻이기도 합니다. 저희는 이걸 완성이 아니라, 0.41에서 0.87로 끌어올린 현실적 출발점으로 봅니다."

**[각주]** DINOv2 v34=0.806 / v24=0.872 (`all_models_ood_v2.json`), 앙상블 0.89 (`ensemble_sweep_all.json`, thr=0.081)

## 슬라이드 8 — XAI: "점수가 아니라 '왜 FAKE인지'를 보여준다" (★ 우리 차별점)

**[화면 묘사]** — 실제 제품 캡처(Solomon AI) 그대로
- **판정**: CRITICAL / FAKE, Deepfake 확률 **92.5%**, 분석 프레임 30
- **공간 근거**: Original Target vs **GradCAM Heatmap** (코·얼굴 중앙에 열점)
- **시간 근거**: Frame-by-Frame Anomaly 그래프 (0~8.48s, threshold 라인)
- **기여도 분해**: 영상마다 RGB(공간) vs 주파수 중 결정 근거가 다름 → 매 판정마다 비율 분해 (예시 이 영상: Freq 63% / Spatial 37%)
- 하단 메시지: **"공간(어디)·시간(언제)·주파수(무엇으로) 3중 근거 = 설명 가능한 포렌식 증거"**
- (B3) 포지셔닝: **과학적 주장이 아니라 제품 UX** — 기자·일반인이 숫자가 아니라 눈으로 납득

## 예상 Q&A (방어용, 슬라이드 X — 머릿속)

| 질문 | 답 |
|------|-----|
| "왜 140개뿐?" | 크롤링 대량수집 시도했으나 **AI생성 여부 검증 안 되면 점수가 무의미** → 양보다 라벨 신뢰성 택함(real70/fake70 직접 검증). 신뢰구간 넓음 인정, 확장이 다음 과제 |
| "leakage 아니냐?" | 맞다, 앙상블 가중치를 같은 셋에서 sweep함. 그래서 **단일 0.872를 대표 수치**로 본다 |
| "왜 frequency냐 / 데이터빨 아니냐" | 데이터빨 맞다(슬6에서 먼저 인정). 주파수는 +0.066 보조 |
| "왜 DINOv2 안 썼나" | 우리 OOD에선 0.81로 더 낮았고 추론 무거움 |
| "학습/평가 분포 겹침?" | DF40은 공개셋, SNS real은 자체수집 → 평가 140과 출처 분리 |
| "XAI 63%와 슬6 모순?" | 63%는 **이 영상 한 건** 기준(샘플마다 다름)인 판정 근거 vs 슬6은 OOD 일반화 주역 — 다른 층위 |
| "RGB/주파수 비율 뭘 기준으로 나누나?" | 임의 아님. 모델 내부 **SE-block 채널 어텐션 게이트**값을 RGB(앞2048)/주파수(뒤2048)로 나눠 평균낸 비율 = 모델이 그 영상 판정에 실제로 둔 가중치. (코드 그대로) |
| "수학적으로 엄밀한 기여도냐?" | 엄밀한 attribution(Shapley/IG) 아님, **모델 내부 어텐션을 직관적으로 보여주는 근사치**. XAI는 과학적 증명이 아니라 설명 보조(제품 UX)로 포지셔닝 |
